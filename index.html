设置一个容器服务的日志大小上限

在启动容器的时候增加一个参数设置该容器的日志大小，及日志驱动

全局设置

1.编辑文件/etc/docker/daemon.json, 增加以下日志的配置

max-size=500m，意味着一个容器日志大小上限是500M，
max-file=3，意味着一个容器有三个日志，分别是id+.json、id+1.json、id+2.json。

2.然后重启docker守护进程

注意：设置的日志大小，只对新建的容器有效。

10.Linux的docker的部署方式：docker run -d -p 9400:9400 -e TZ=’Asia/Shanghai’ -v /usr/local/tinyDev:/data –name dev_tiny_gateway java:8u111 java -Xms1024m -Xmx1024m -jar /data/tiny_gateway/lib/tiny-gateway.jar –spring.config.location=file:/data/tiny_gateway/conf/

- /usr/local/tinyDev：容器需要映射出来的服务器地址
- dev_tiny_gateway：需要取的名字
- java:8u111：jdk镜像
- -Xms1024m -Xmx1024m：堆栈大小
- /data/tiny_gateway/lib/tiny-gateway.jar：jar包位置
- file:/data/tiny_gateway/conf/：配置文件所在位置
